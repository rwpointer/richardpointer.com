Two-Hall Configuration

Complete equipment checklist and setup protocol for running 16 electronic boards across two simultaneous tournament halls.

16 DGT Boards 2 Tournament Halls 2 Host Computers 8 Boards per Hall
01

Background

Why Two Computers?

DGT LiveChess — the software that reads moves from DGT electronic boards and broadcasts them — communicates with boards over USB or serial connections. Because USB cables have a practical length limit and a single host can become a single point of failure, splitting boards across two physically separate halls requires a dedicated host computer in each hall.

Hall A

Computer A

Runs DGT LiveChess for boards 1–8. Broadcasts to the local network and/or internet. The arbiter designates this as the primary instance for livegame feed aggregation.

Hall B

Computer B

Runs a second instance of DGT LiveChess for boards 9–16. Uses a different port (or separate LiveChess server endpoint) to avoid conflicts. Broadcasts on the same network.

⚠ Critical constraint

DGT LiveChess only exposes boards physically connected to the machine it's running on. There is no built-in USB sharing across a network. Each hall must have its own dedicated machine — a single laptop cannot serve both rooms.

ℹ Port configuration

By default, DGT LiveChess listens on port 1982. If both computers are on the same network and a single downstream aggregator (e.g., a relay script or a Chess Arbiter integration) needs to poll both, configure Computer B to use port 1983 — or use the LiveChess server settings to bind to a different hostname/port. Confirm this with your broadcast partner before the event.

Day-of Protocol

Step-by-Step Setup

📋 Pre-event requirement

Test the full 16-board USB chain and both LiveChess instances at least one day before the tournament. Do not assume cables and boards that worked last time will work this time. Verify board serial number recognition in the LiveChess interface.

  1. 1
    Position boards on tables before powering anything

    Place all 16 boards first. Route USB cables under or along table edges. Boards 1–8 in Hall A, 9–16 in Hall B. Do not plug in yet — lay cables cleanly to avoid tripping hazards.

  2. 2
    Connect boards to USB hubs, one hall at a time

    In Hall A: connect boards 1–8 to the 8-port powered hub. In Hall B: boards 9–16 to Hub B. Label each port on the hub with a strip of tape matching the board number if needed.

  3. 3
    Power on hubs, then connect to computers

    Always power the hub before connecting to the PC. This reduces device enumeration errors. Plug Hub A into Computer A and Hub B into Computer B via a single USB-A cable from hub to machine.

  4. 4
    Launch DGT LiveChess on both computers

    Open LiveChess on Computer A first. Let it enumerate boards — you should see 8 boards appear. Repeat on Computer B. If a board doesn't appear, swap the cable first before assuming a board failure.

  5. 5
    Assign board numbers and configure the board layout

    In LiveChess, drag-and-drop or manually assign board numbers 1–8 (Hall A) and 9–16 (Hall B) to match the physical board positions. Export the .lc configuration file as a backup.

  6. 6
    Test each board with a known position

    Place a few pieces on each board and confirm the position is reflected in the LiveChess interface. This catches dead boards, loose connectors, and piece sensor failures before players sit down.

  7. 7
    Start the broadcast / relay connection

    Connect both LiveChess instances to your broadcast aggregator. If using a relay script, point it at both Computer A (:1982) and Computer B (:1983). Confirm games are appearing live in the broadcast interface with correct board assignments.

  8. 8
    Brief the arbiters in each hall

    Each hall arbiter should know: which computer serves their boards, how to check the LiveChess interface for status, and the fallback protocol (paper scoresheet) if a board goes offline mid-game.

Reference

Common Problems

  • Board not recognized in LiveChess
    Swap USB cable first. Then try a different hub port. If still failing, plug the board directly into the computer (bypassing the hub) to isolate the issue. Reinstall DGT driver if needed.
  • Boards drop out intermittently
    Almost always a power issue with the USB hub. Confirm the hub's power adapter is connected and supplying adequate amperage. Avoid connecting non-DGT devices to the same hub.
  • Wrong moves appearing on broadcast
    Board numbers in LiveChess don't match physical positions. Recheck board number assignment and the pairing assignment in your tournament management software. Pieces on the wrong squares can also cause phantom moves — reseat all pieces.
  • LiveChess not broadcasting to internet
    Check firewall settings on both computers — ports 1982/1983 must be open. Verify the relay script or broadcast API key. If on a venue network, confirm the IT team has not blocked outbound traffic on those ports.
  • One entire hall goes dark mid-round
    Usually a power failure to the hub or the computer. Switch to paper scoresheets immediately and do not stop play. Restart LiveChess after power is restored — it will re-enumerate boards and you can resume broadcasting the remaining moves once reconnected.
